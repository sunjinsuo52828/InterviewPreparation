Webpack Core Concepts

Webpack is a static module bundler for modern JavaScript applications.

ğŸšª Entry

The entry point indicates which module webpack should use to begin building out its internal dependency graph.

entry: './src/index.js'

ğŸ“¦ Output

Tells webpack where to emit the bundles it creates and how to name these files.

output: { filename: 'bundle.js' }

ğŸ”„ Loaders

Webpack only understands JS and JSON. Loaders allow webpack to process other types of files (CSS, Images, TS).

test: /\.css$/, use: 'css-loader'

ğŸ”Œ Plugins

Plugins perform a wider range of tasks: bundle optimization, asset management, environment injection.

âš™ï¸ Mode

Set development, production or none to enable webpack's built-in optimizations.

ğŸ¤ High Frequency Interview Questions

1. Loader vs Plugin? Must Know

Loader: Transforms individual files. Works at module level.
Example: babel-loader (JSâ†’JS), css-loader (CSSâ†’JS)

Plugin: Works on the entire compilation lifecycle. Does everything loaders can't.
Example: HtmlWebpackPlugin, DefinePlugin, MiniCssExtractPlugin

2. How to improve Webpack Build Speed? Performance
  • Update Webpack/Node: Newer versions are faster.
  • Loader Scope: Use include/exclude (skip node_modules).
  • Caching: cache: { type: 'filesystem' } (Webpack 5).
  • Parallelism: thread-loader for expensive loaders.
  • Dev vs Prod: Skip minification/sourcemaps in dev.
3. How does HMR (Hot Module Replacement) work? Principle

1. Webpack Dev Server watches files, builds in memory.

2. On change, WDS sends hash to browser via WebSocket.

3. Browser runtime requests updated manifest + chunks (JSONP).

4. Runtime replaces old module with new one without full reload.

4. Webpack vs Vite? Trending

Webpack: Bundle-based. Builds entire dependency graph first. Rich ecosystem, complex configs.

Vite: Native ESM in dev (no bundling). Extremely fast cold start and HMR. Uses Rollup for production.

When to use Vite? New projects, modern browsers. Webpack? Legacy support, complex builds.

5. What is Tree Shaking? Optimization

Dead code elimination based on ES Module static analysis.

Requirements:

  • Use ES Modules (import/export), not CommonJS (require)
  • Set mode: 'production'
  • Set sideEffects: false in package.json (or list side-effect files)
6. What is Module Federation? Webpack 5

Allows multiple independent builds to share code at runtime.

Use Case: Micro-Frontends - different teams deploy separate apps that compose into one.

Key Concepts:

  • exposes: Modules this build shares
  • remotes: Modules consumed from other builds
  • shared: Common dependencies (React, etc.)
