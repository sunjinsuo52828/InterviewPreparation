How it works:

The Thread Pool is just a container of Worker Threads.

The logic is encapsulated in the Runnable / Callable object (the Task), not the Thread itself.

// Thread's internal loop (Simplified)
while (true) {
    Runnable task = queue.take(); // Get ANY task
    task.run(); // Execute the task's specific logic
}

So, Thread T1 can execute a Math calculation now, and a Database query 1 second later.
