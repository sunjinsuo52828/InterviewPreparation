Q1: Why are there two Survivor spaces (S0/S1)?
To solve Memory Fragmentation. By copying live objects between S0 and S1, we ensure that the Survivor space is always contiguous. If we only had one, clearing dead objects would leave holes (fragmentation).
Q2: When does an object enter Old Gen?
1. Age Threshold: Usually 15 (CMS/Serial) or dynamic (G1).
2. Survivor Overflow: If Survivor space is full, objects go directly to Old Gen.
3. Big Objects: Large arrays/objects may skip Young Gen entirely (Humongous Objects in G1).
Q3: What is the difference between G1 and CMS?
CMS: Mark-Sweep (Old Gen). Low latency but causes fragmentation (needs Full GC to compact).
G1: Region-based + Copying. Compacts while collecting. Predictable pause times.
Q4: What causes Metaspace OOM?
Loading too many classes! Common in frameworks using CGLib/Dynamic Proxies (Spring, Hibernate) that generate classes at runtime. Unlike PermGen, it uses native memory, so it can eat up all server RAM if not capped.
Q5: What are GC Roots?
Starting points for reachability analysis. Includes: Stack Variables (Local vars), Static Variables, JNI References, and active Threads.
Q6: What is G1NewSizePercent?
It sets the Minimum Young Gen Size (Default 5%). G1 is adaptive: it shrinks Young Gen to meet MaxGCPauseMillis. This flag acts as a "safety floor" to prevent Young Gen from becoming too small, which would cause frequent, inefficient GCs.
