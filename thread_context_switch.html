Q1: What is a Context Switch?
It's the process of storing the state (Context) of the currently running thread so that it can be resumed later, and restoring the state of a different thread to run.
Q2: What exactly is saved in the Context?
The TCB (Thread Control Block) stores:
  • Program Counter (PC): The address of the next instruction to execute.
  • Registers: General purpose registers (RAX, RBX...), Stack Pointer (SP).
  • Thread State: Running, Ready, Blocked.
Note: It does NOT save the entire Heap (shared). It saves the Stack Pointer, which points to the thread's private stack.
Q3: What is the "Cost" of Context Switching?
  1. Direct Cost: CPU cycles spent saving/loading registers (overhead).
  2. Indirect Cost (The Big One): Cache Pollution. The new thread likely needs different data, causing L1/L2 cache misses and TLB (Translation Lookaside Buffer) flushes.
Q4: When does it happen?
  • Time Slice Expiry: Preemptive multitasking (OS says "Time's up!").
  • I/O Blocking: Thread waits for disk/network.
  • Higher Priority: A more important thread wakes up.
